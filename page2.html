💻软件

宠物喂食器配套软件使用说明和开发文档

软件架构

本项目的软件系统分为三个部分：嵌入式程序、后端服务和移动APP。各部分相互协作，实现完整的智能喂食功能。

🔧

嵌入式程序

运行在ESP32主控板上的固件程序，负责硬件控制、传感器数据采集、通信协议处理等功能

🖥️

后端服务

基于Node.js开发的服务端，提供RESTful API，处理数据存储、消息推送、设备管理等业务逻辑

📱

移动APP

使用Flutter开发的跨平台应用，提供友好的用户界面，支持iOS和Android双平台

环境要求

  • Node.js - v16.0.0 或更高版本
  • Python - v3.8 或更高版本
  • Arduino IDE - v2.0 或更高版本（用于烧录固件）
  • Flutter SDK - v3.0 或更高版本（用于APP开发）

快速开始

编译烧录固件

# 克隆固件仓库 git clone https://gitee.com/your-repo/pet-feeder-firmware.git cd pet-feeder-firmware # 使用PlatformIO编译 pio run # 烧录到设备 pio run --target upload

启动后端服务

# 克隆后端仓库 git clone https://gitee.com/your-repo/pet-feeder-server.git cd pet-feeder-server # 安装依赖 npm install # 启动服务 npm run start

运行移动APP

# 克隆APP仓库 git clone https://gitee.com/your-repo/pet-feeder-app.git cd pet-feeder-app # 安装依赖 flutter pub get # 运行调试 flutter run

配置说明

在启动服务前，需要配置相关参数。复制配置文件并修改对应内容：

# 配置文件 config.json { "server": { "port": 3000, "host": "0.0.0.0" }, "database": { "type": "sqlite", "path": "./data/pet-feeder.db" }, "mqtt": { "broker": "mqtt://localhost:1883", "username": "admin", "password": "password" } }
⚠️ 注意： 生产环境请修改默认密码并使用更安全的认证方式！

API接口

后端服务提供以下主要API接口：

接口 方法 描述
/api/device/status GET 获取设备状态
/api/feed POST 触发喂食
/api/schedule GET/POST 定时喂食计划
/api/stats GET 获取喂食统计
